# Ручные тест-кейсы PlaneFlow (для начинающего тестировщика)

Это набор **ручных** тест-кейсов для игры PlaneFlow (аэропорт-диспетчер).
Писалось максимально просто — даже если ты тестируешь впервые, разберёшься.

## Зачем это нужно
Когда в игру добавляют что-то новое или чинят баги, нужно проверить, что **всё
старое не сломалось** и **новое работает**. Тест-кейс — это пошаговая инструкция:
что нажать, что должно произойти. Ты идёшь по шагам и сравниваешь результат.

## Как пользоваться (важно прочитать новичку)

1. Открой игру: файл `index.html` в браузере, либо запусти локальный сервер
   `python3 -m http.server 8000` и зайди на `http://localhost:8000`.
   Прод-версия — на GitHub Pages (см. README проекта).
2. Игра **горизонтальная** — держи телефон ландшафтно (поверни вбок).
3. Бери тест-кейсы по одному. Внутри каждого:
   - **Предусловие** — что должно быть готово до начала (например, «открыта игра,
     идёт 1-й уровень»).
   - **Шаги** — что делать по порядку.
   - **Ожидаемый результат** — что должно произойти. Если совпало — тест
     **пройден (PASS)**. Если нет — **провален (FAIL)**, и это баг.
4. Если нашёл баг — запиши: **что делал, что ожидал, что получилось** (и приложи
   скриншот/видео, если можешь). Этого достаточно для заведения бага.

## Полезные слова (глоссарий)

- **Борт** — самолёт.
- **ВПП / полоса** — взлётно-посадочная полоса (справа на экране).
- **Бокс** — сервисная стоянка по краям поля (ремонт / топливо / посадка людей).
- **Терпение** — колечко вокруг борта; когда кончается, борт злится/бьётся.
- **Жесть/проводка пальцем** — провести пальцем (или мышкой) от борта до цели,
  рисуя линию пути.
- **Тап** — короткое нажатие.
- **HUD** — верхняя панель с деньгами, жизнями, счётом и т.д.
- **Комбо/серия** — множитель денег за вылеты подряд без аварий.
- **PWA** — «приложение из браузера»: игру можно установить на телефон и играть
  офлайн.

## Где искать настройки для тестов

В **паузе → Настройки** есть отладочные галочки:
**бесконечные жизни** и **безлимит денег**. Включай их, когда нужно спокойно
проверить механику, не отвлекаясь на проигрыш. Там же — язык, звук, вибрация,
сброс прогресса.

## Структура папки

- [`01-onboarding-menu.md`](01-onboarding-menu.md) — главное меню, выбор уровня, старт.
- [`02-core-mechanics.md`](02-core-mechanics.md) — базовая механика: посадка, боксы, вылет, деньги.
- [`03-patience-lives-collisions.md`](03-patience-lives-collisions.md) — терпение, жизни, столкновения, проигрыш.
- [`04-special-planes-events.md`](04-special-planes-events.md) — спец-борты (VIP, аварийный, медицинский) и события.
- [`05-levels-stars-progress.md`](05-levels-stars-progress.md) — уровни, звёзды, сохранение, бонус-уровни, Zen.
- [`06-pause-settings-menu.md`](06-pause-settings-menu.md) — пауза, настройки, язык, звук, вибрация.
- [`07-pwa-browser.md`](07-pwa-browser.md) — установка PWA, офлайн, обновления, поведение в браузере.
- [`08-timings.md`](08-timings.md) — тайминги и числа (что за сколько срабатывает).

> Числа в кейсах (терпение, цены, время) взяты из конфига `K` в `src/game/04-config-levels.ts`.
> Если в игре поменяли баланс — обнови ожидаемые значения в кейсах.
