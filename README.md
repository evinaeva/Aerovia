# PlaneFlow

Мобильная игра — **аэропорт-диспетчер**. Борта прилетают, ты пальцем сажаешь их на
полосу, развозишь по боксам обслуживания (ремонт / топливо / посадка людей) в нужном
порядке и отправляешь на взлёт. Следи за жизнями, терпением бортов и не дай им
столкнуться. Деньги тратятся на открытие и апгрейд боксов; есть уровни и сохранение.

HTML5 Canvas + ванильный JS, без рантайм-зависимостей. Исходник **модульный** (в
`src/`); `npm run build` склеивает его в один самодостаточный `index.html` (см. «Сборка»).

Облик — **neon** (спокойный ночной радар: тёмная база, бирюзово-розовое свечение):
игра рисуется векторными спрайтами дизайн-системы PlaneFlow из `assets/sprites/`
(бренд / самолёты / поле / HUD / эффекты) через спрайт-атлас с процедурным
фолбэком (флаг `ATLAS` в `index.html`). Спека спрайтов — `assets/sprites/README.md`.
Облик один — выбора скина нет.

Языки: **английский** (основной) и **русский** — автоопределение по устройству,
переключатель в настройках (пауза → Настройки). Новые языки добавляются словарём
(см. `docs/DEV.md`, раздел «Локализация»).

## Сборка

`index.html` — **генерируемый файл** (в git его нет). Собрать из `src/`:

```sh
npm run build     # node scripts/build.mjs → собирает index.html из src/
```

Сборка только склеивает модули и вставляет CSS/JS в каркас `index.template.html` —
без минификации и транспиляции, на выходе обычный читаемый HTML. Редактируй **`src/`**,
не `index.html`.

## Запуск

Сначала `npm run build`, затем открыть `index.html` в браузере. Для теста на телефоне
удобнее локальный сервер (`npm run serve` сам пересобирает и поднимает раздачу):

```sh
npm run serve     # сборка + python3 -m http.server 8123
# затем открыть http://<ip-компьютера>:8123 на телефоне в той же сети
```

Прод-сайт — **GitHub Pages**, публикуется **роботом** (GitHub Actions,
`.github/workflows/deploy.yml`): на каждый push в `main` он собирает `index.html` из
`src/` и выкладывает на свой домен **`https://planeflow.jevgenia.com`** (файл `CNAME`).
Игра горизонтальная — держи телефон ландшафтно.

**Рабочий процесс:** работаем в **своей feature-ветке** (не коммитим прямо в `main`):
ответвись от свежего `origin/main`, сделай правки **в `src/`**, запушь ветку и открой PR.
**В `main` сами не мержим** — мёрж делает владелец. Часто параллельно идёт несколько
диалогов Claude по этому репо, и авто-мёрж в `main` сбивает их работу. После мёржа робот
сам пересобирает и публикует сайт — затем жёсткий рефреш (Ctrl+Shift+R), т.к. страница
кэшируется.

## Структура

- `src/` — **исходник игры** (редактируй здесь): `styles.css`, `game/01..13-*.js`
  (склеиваются в один скрипт по порядку), `boot-sw.js`; `index.template.html` — каркас.
- `scripts/build.mjs` — сборка `index.html` из `src/`. Сам `index.html` —
  **генерируемый артефакт**, в git его нет, руками не редактируется.
- `manifest.json` — web-app манифест (имя, иконки, ландшафт, standalone) для установки PWA.
- `sw.js` — service worker: офлайн-старт + контролируемые обновления. После
  изменения precache-списка (или чтобы форсить обновление у игроков) подними
  `VERSION` в `sw.js` — игрок увидит тост «Доступна новая версия → Обновить».
- `assets/sprites/` — канонические спрайтшиты дизайн-системы (+ neon-арт в `neon/`).
- `assets/icon/` — иконка приложения: 512 + Android adaptive (bg/fg).
- `docs/design/` — **единый дизайн-хаб**: арт-дирекшн, геймдизайн, скины, wireframes,
  референсы, рабочие материалы (см. `docs/design/README.md`).
- `docs/` — бриф, FAQ, дев-доки и прочая документация.
- `tests/` — тесты (страховка под новые механики), `package.json` — раннеры.
- `CHANGELOG.md` — версии.

## Тесты

Страховка от регрессий при добавлении механик/событий. Подробности — в
[`docs/DEV.md`](docs/DEV.md), раздел «Тесты».

```sh
npm test          # юнит-логика + валидатор конфига (Node, без браузера)
npm run test:e2e  # e2e-смоук (Playwright; нужен: npx playwright install chromium)
```
