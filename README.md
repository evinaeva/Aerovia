# PlaneFlow

Мобильная игра — **аэропорт-диспетчер**. Борта прилетают, ты пальцем сажаешь их на
полосу, развозишь по боксам обслуживания (ремонт / топливо / посадка людей) в нужном
порядке и отправляешь на взлёт. Следи за жизнями, терпением бортов и не дай им
столкнуться. Деньги тратятся на открытие и апгрейд боксов; есть уровни и сохранение.

HTML5 Canvas + ванильный JS, один файл, без зависимостей.

Облик — **cozy / stylized-flat** (тёплая ночь, navy/purple, кремовые борта): игра
рисуется векторными спрайтами дизайн-системы PlaneFlow из `assets/sprites/`
(бренд / самолёты / поле / HUD / эффекты) через спрайт-атлас с процедурным
фолбэком (флаг `COZY` в `index.html`). Спека спрайтов — `assets/sprites/README.md`.

**Скины:** три темы оформления — **neon** (спокойный ночной радар, *по умолчанию*),
**cozy** (тёплая ночь) и **cartoon** (яркий день). Переключаются в **Настройках →
Скин** (или `?skin=<name>` / `localStorage.pf_skin`), различаются только **видом**, не
правилами (см. `docs/design/skins/README.md`). Главный экран собран в три финальные
раскладки — по одной на скин — на одной разметке через `[data-skin]`.

Языки: **английский** (основной) и **русский** — автоопределение по устройству,
переключатель в настройках (пауза → Настройки). Новые языки добавляются словарём
(см. `docs/DEV.md`, раздел «Локализация»).

## Запуск

Открыть `index.html` в браузере. Для теста на телефоне удобнее поднять локальный сервер:

```sh
python3 -m http.server 8000
# затем открыть http://<ip-компьютера>:8000 на телефоне в той же сети
```

Прод-сайт — **GitHub Pages** из ветки `main` (Settings → Pages → ветка `main`, корень)
на своём домене **`https://planeflow.jevgenia.com`** (файл `CNAME`). Игра горизонтальная —
держи телефон ландшафтно.

**Рабочий процесс / деплой:** коммитим и **пушим сразу в `main`** — GitHub Pages
публикует автоматически (отдельные PR-ветки не обязательны). После пуша делай
жёсткий рефреш (Ctrl+Shift+R), т.к. `index.html` кэшируется.

## Структура

- `index.html` — вся игра (Canvas + JS).
- `manifest.json` — web-app манифест (имя, иконки, ландшафт, standalone) для установки PWA.
- `sw.js` — service worker: офлайн-старт + контролируемые обновления. После
  изменения precache-списка (или чтобы форсить обновление у игроков) подними
  `VERSION` в `sw.js` — игрок увидит тост «Доступна новая версия → Обновить».
- `assets/sprites/` — канонические спрайтшиты дизайн-системы (+ пер-скиновый арт).
- `assets/icon/` — иконка приложения: 512 + Android adaptive (bg/fg).
- `docs/design/` — **единый дизайн-хаб**: арт-дирекшн, геймдизайн, скины, wireframes,
  референсы, рабочие материалы (см. `docs/design/README.md`).
- `docs/` — бриф, FAQ, дев-доки и прочая документация.
- `tests/` — тесты (страховка под новые механики), `package.json` — раннеры.
- `CHANGELOG.md` — версии.

## Тесты

Страховка от регрессий при добавлении механик/событий. Подробности — в
[`docs/DEV.md`](docs/DEV.md), раздел «Тесты».

```sh
npm test          # юнит-логика + валидатор конфига (Node, без браузера)
npm run test:e2e  # e2e-смоук (Playwright; нужен: npx playwright install chromium)
```
