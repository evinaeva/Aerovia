## Что это
Мобильная игра в жанре «диспетчер: веди маршруты пальцем» (как *Airport Control* /
*Flight Control*). Игрок ведёт самолёты пальцем к посадочной полосе нужного цвета,
не допуская столкновений в воздухе.

**Ощущение, ради которого всё затевается:** спокойный поток / лёгкая медитация —
мозг занят распределением задач и отдыхает от реальности. Это не агрессивная аркада.
Любое решение по балансу и эстетике сверяем с этим ощущением.

**Двойная цель:**
1. Своя игра, в которую приятно играть и бесконечно добавлять уровни.
2. Публикация в Google Play.

## Роли
- **Продукт-овнер / геймдизайн / баланс / идеи уровней** — человек.
- **Технический исполнитель (весь код)** — ИИ (Claude Code).
  Человек не пишет код, а формулирует, что должно происходить и как ощущаться.

## Юридическая рамка
- Игровая механика и идея жанра **не защищены** авторским правом — делать свою игру
  с той же сутью законно.
- **Нельзя** копировать чужое конкретное: название «Airport Control», её графику,
  звуки, тексты, иконку.
- Используем своё имя, свой арт, свои ассеты.

## Текущее состояние
Есть рабочий прототип: один файл **`flight_tower_prototype.html`**
(HTML5 Canvas, ванильный JS, без зависимостей). Открывается в любом браузере и на телефоне.

**Что уже работает:**
- Спавн самолётов с краёв экрана, 3 цвета.
- Рисование маршрута пальцем/мышью (pointer + touch события).
- Движение по маршруту с ограничением скорости поворота → плавные дуги.
- Отскок от краёв, когда маршрут не задан (борт остаётся в игре).
- Посадка на полосу совпадающего цвета (+ счёт).
- Столкновение двух бортов = конец смены.
- Нарастание сложности (частота спавна и скорость растут со счётом).
- Экраны старта и конца смены, рестарт.
- Эстетика: спокойный «радар ночного аэродрома», палитра amber / teal / rose,
  моноширинный HUD, мягкое свечение траекторий.

**Ключевые «крутилки» баланса в коде** (искать по именам):
- `TURN` — макс. скорость поворота борта (плавность дуг).
- `CRASH_DIST` — на каком расстоянии считается столкновение.
- `GRAB` — радиус захвата самолёта пальцем (важно для мобилки).
- `speed` / `interval` спавна — кривая сложности.
- `RUNWAY_DEFS` — положение и цвет полос (заданы долями экрана).

## Технологический план (порядок шагов)
1. **Разработка** — в браузере. Самый быстрый цикл: правка кода → результат за секунды.
2. **Проверка ощущения** — на реальном телефоне (через GitHub Pages или локальный сервер).
   Настоящий палец важнее эмулятора; эмулятор Android Studio на этом этапе не нужен.
3. **«Как приложение» без сборки** — PWA: хостинг (GitHub Pages / Netlify / itch.io)
   + «Добавить на главный экран». Иконка, полный экран, почти неотличимо от приложения.
4. **Публикация в Google Play** — завернуть PWA в Android-пакет `.aab`:
   - **PWABuilder** — почти кнопочный путь (дать ссылку на игру → получить пакет).
   - **Capacitor** — мощнее, но требует установки инструментов (Node, Android SDK).

## Сохранение прогресса
- **Наработки проекта** — Git / GitHub (этот репозиторий).
- **Внутриигровое сохранение** (рекорд, открытые уровни, настройки) —
  `localStorage` или `IndexedDB`.
  ⚠️ Примечание: `localStorage` **не работает** внутри артефактов Claude.ai, но
  прекрасно работает в обычной веб-странице / на хостинге / в Claude Code.
  В «настоящем» проекте это правильный путь, ограничение было только у предпросмотра в чате.

## Google Play — факты (актуально на июнь 2026)
- **$25 — разовый сбор за аккаунт**, не за игру. Под одним аккаунтом можно
  публиковать сколько угодно игр. Нет ежегодных / помесячных / по-приложению платежей.
- Нужна **проверка личности** (госдокумент).
- Для **личного (Personal) аккаунта** — обязательное закрытое тестирование с
  **12 тестировщиками ~2 недели** до открытой публикации. Заложить время и людей заранее.

## Монетизация (когда дойдём до этого этапа)
- **AdMob** — бесплатный рекламный модуль; Google делится доходом от рекламы,
  разработчик ему ничего не платит.
- Ориентиры eCPM (доход за 1000 показов, начало 2025):
  баннеры ~$1.5; межстраничные $3–6 (до ~$13 в богатых регионах);
  **наградное видео $15–25** — самый прибыльный формат. Сильная зависимость от страны игроков.
- Реалистичный масштаб: ~10k активных игроков/мес ≈ $50–500/мес.
  Чисто на рекламе нишевая игра — сначала «на кофе»; деньги начинаются с объёма аудитории.
- **Лучший фит для этой игры — наградная реклама:** «разбил борта — посмотри ролик,
  продолжи смену» / «посмотри рекламу — открой бонусный набор уровней». Игрок выбирает сам,
  не раздражает, платит больше всего.
- Дополнительно: разовая покупка «убрать рекламу», платные наборы уровней.
- Базовая игра бесплатная — для набора аудитории.
- При интеграции важно: точки показа рекламы не должны убивать медитативное ощущение.

## Бэклог / приоритеты

**Ближайшее — довести прототип до «не стыдно показать»:**
1. Докрутить ощущение управления (повороты `TURN`, скорости, отзывчивость захвата `GRAB`).
2. Звук и тактиль: сигнал «принят борт», мягкий звук посадки, вибрация.
3. **Система уровней:** уровень = конфиг (расположение полос, число цветов,
   частота/скорость спавна, спец-условия — запретные зоны, погода и т.п.). Прогрессия.
4. Внутриигровое сохранение (рекорд, открытые уровни) — `localStorage`.
5. Меню выбора уровней.

**Потом:**
6. PWA-обёртка + хостинг, тест на телефоне.
7. Иконка, скриншоты, название, описание для стора.
8. Упаковка в `.aab`, закрытое тестирование (12 человек), публикация.
9. Интеграция AdMob (наградная реклама в первую очередь) + «убрать рекламу».
