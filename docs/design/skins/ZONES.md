# Aerovia — Zone Skin Spec

> **Prompt for Claude Code.** Follow this file literally to add or update zone skins.
> Every section is a task, not a suggestion.

---

## What «a skin» is

One skin = one folder under `assets/skins/<zone>/<name>/` with:
- `skin.json` — metadata + file list
- one or more `.png` files — the actual art

The tuning workbench (`tuning.html` → **Скины**) reads `assets/skins/index.json`
(generated by `npm run scan:skins`) and shows all registered skins as chips per zone.
The player picks a chip, the canvas shows the skin on real zone geometry.

## skin.json format

```json
{
  "id": "hangar-neon",
  "label": "Human-readable name shown in the tab",
  "states": {
    "state-key": "filename.png"
  }
}
```

Rules:
- `id` — stable unique string per zone, used in level export JSON. Never change it after first use.
- `label` — shown as chip text in the Скины tab.
- `states` — keys must match exactly what the engine expects for this zone (see per-zone tables below).
- `"base"` (optional) — if set, image paths resolve from repo root relative to that base folder instead of the skin folder. Use to reuse existing sprites without copying.
- If a state file is missing on disk, `scan-skins.mjs` warns and skips it.

## Zone specs

### 1. `hangar` — Ангар

State keys the engine uses:

| key | when drawn |
|-----|------------|
| `fuel` | bay open, fuel service |
| `board` | bay open, boarding service |
| `repair` | bay open, repair service |
| `deice` | bay open, de-icing service |
| `locked` | bay locked / not purchased |

The engine draws ON TOP: service icon, price tag, upgrade pips, progress ring, parked plane.
**Do not bake any of these into the PNG.**

PNG spec:
- Canvas: **256 × 256 px**, transparent background
- Content: square bay panel occupying ~80 % of the canvas, centered
- Gate (open side) must face **down** — the engine rotates the sprite for top-row hangars
- Leave center area clear for the parked plane

Visual (neon theme): dark panel, neon-coloured border per state: teal (fuel), rose (board), amber (repair), ice-blue (deice), muted grey (locked).

---

### 2. `apron` — Апрон

One state, any key name (engine uses the first state only).

```json
{ "id": "apron-01", "label": "Апрон 01", "states": { "default": "apron.png" } }
```

PNG spec:
- Canvas: **512 × 512 px**, transparent background
- Content: floor material + border/edge lights only — the centre must be empty (planes taxi there)
- The engine stretches this to fill whatever apron rectangle the layout defines

Visual: dark tarmac / concrete texture, neon edge lighting.

---

### 3. `runway` — ВПП

One state, any key name (engine uses the first state only).

```json
{ "id": "runway-01", "label": "ВПП 01", "states": { "default": "runway.png" } }
```

PNG spec:
- Canvas: **512 × 192 px**, transparent background
- Content: horizontal strip — end caps + seamless centre section; runway markings baked in
- Width (height on screen) = 1.23 × PLANE_LEN at runtime; the engine stretches the strip

Visual: dark strip, white centre-line dashes, neon threshold lights.

---

### 4. `plane` — Самолёт

One state, any key name (engine uses the first state only).

```json
{ "id": "plane-01", "label": "Ливрея 01", "states": { "default": "plane.png" } }
```

PNG spec:
- Canvas: **256 × 256 px**, transparent background
- Content: top-down airliner, **nose pointing UP**, centred
- Engine draws on top: selection ring, patience ring, service icon — do not bake

Visual: light body, neon outline + glow for neon theme.

---

### 5. `arrival` — Прилёт

One state, any key name.

```json
{ "id": "arrival-01", "label": "Прилёт 01", "states": { "default": "arrival.png" } }
```

PNG spec:
- Canvas: **512 × 512 px**, transparent background
- Content: rectangular landing zone marker, floor texture — centre must stay clear

---

### 6. `background` — Фон

One state, any key name.

```json
{ "id": "bg-01", "label": "Фон 01", "states": { "default": "background.png" } }
```

PNG spec:
- Canvas: **1920 × 1080 px** (or any 16:9 ratio), no transparency required
- Content: full field background — night sky / airfield panorama

---

## What to bake into PNG vs. leave for the engine

| Bake ✅ | Do NOT bake ❌ |
|---------|---------------|
| Glow, neon border, gloss | Service icon |
| Gradients, soft shadow | Price tag |
| Floor texture, wall material | Upgrade pips |
| Gate shape, runway markings | Progress ring |
| Lighting effects | Parked plane inside hangar |

## How to register skins (required steps after adding files)

1. Place all PNG files under `assets/skins/<zone>/<name>/`.
2. Write `skin.json` in each folder.
3. Run: `npm run scan:skins`
   - This regenerates `assets/skins/index.json`.
   - Warnings mean a listed PNG is missing — fix before continuing.
4. Commit both the skin folder and the updated `assets/skins/index.json`.
5. Verify in the workbench: `npm run serve` → open `tuning.html` → **Скины** tab → skin chips must appear for each zone.

## Definition of done

- [ ] Each zone folder exists: `assets/skins/<zone>/<name>/skin.json` + all PNG files present on disk.
- [ ] `skin.json` has a unique `id`, readable `label`, and all required state keys for the zone.
- [ ] `npm run scan:skins` runs without warnings.
- [ ] `assets/skins/index.json` updated and committed.
- [ ] Tuning workbench shows the new skin chips in **Скины** tab.
- [ ] Selecting a skin in the workbench renders it on the canvas preview without errors.
- [ ] Hangar panels: centre area clean — no baked icons, prices, pips, or plane.
- [ ] Plane sprite: nose up, centred in 256 × 256.
