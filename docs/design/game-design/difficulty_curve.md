# PlaneFlow — Кривая сложности

> Дизайн-документ о том, **как растёт сложность** по ходу кампании. Схема конфига уровня и

## Хребет — кампания из отдельных уровней

Сложность PlaneFlow живёт на **лесенке кампании** — это массив `LEVELS` (сегодня 10 уровней,
список расширяемый). Каждый уровень — самодостаточная смена со своей целью и порогами звёзд;
сложность задаётся **на каждый уровень отдельно**, а не как один непрерывный мир. Это
канонная структура. Модель из MVP «единый аэропорт + погода» (см. [`mvp_plan.md`](mvp_plan.md))
**вписана внутрь кампании** как порядок появления новых «игрушек», а не как конкурирующий режим.

Почему отдельные уровни: каждая смена — чистая учебная единица («одна новая идея»), даёт
приятный момент победы (звёзды) и позволяет настраивать одну карту, не задевая остальную кривую.

## Главный принцип — одна новая идея на уровень

Сложность растёт **по одному понятию за раз**. Каждый уровень добавляет одну механику поверх
предыдущего, а число бортов мягко подрастает. Это несущее правило всей кривой — никогда не
сваливай две новые идеи на один уровень.

- **Обучающий блок (L1–L6)** — чистые механики, **без спецсобытий** (это проверяет
  `validateLevels()`): посадка/обслуживание/взлёт, открытие бокса, апгрейд, рост апрона
  (apron — лётное поле со стоянками), таймерный спринт, бóльший ровный поток.
- **Лесенка событий (L7–L10)** — спецборты вводятся поверх крепких механик: VIP, час пик
  (волны бортов), апгрейды под давлением, потом capstone — финальный уровень-итог, где всё
  сходится разом (медицинский + VIP + час пик).

Текущая разбивка по уровням — в таблице в
[`level-pattern.md`](level-pattern.md#the-curve-one-new-idea-per-level).

## Из чего состоит сложность (ручки настройки)

Сложность уровня — это **взвешенная сумма** факторов, как в классике жанра тайм-менеджмента
(Diner Dash и др.), где сложность складывается из плотности прилётов + разнообразия запросов
+ «трудных клиентов» + давления времени. Дизайнер крутит эти ручки; функция `levelDifficulty(lv)`
сводит их в один балл в диапазоне `~[0, ECON_DIFF_CAP]` (этот балл заодно задаёт щедрость
выплат — см. [`economy.md`](economy.md)):

| Ручка | Где | Влияние на кривую |
| --- | --- | --- |
| **Число бортов** (`stars[s1..s3]`) | конфиг уровня | Главный регулятор. Растёт ~+2–4 между порогами 1★; никаких скачков. |
| **Спред звёзд** (`s3 − s1`) | конфиг уровня | Небольшой (1–4): 1★ достижима, 3★ — вызов. Шире = выше потолок мастерства. |
| **Плотность потока** (`flow`) | вычисляется из числа/времени | Чем гуще апрон, тем труднее. Ориентир `ECON_FLOW_REF` = 32. |
| **Давление времени** | `time` / `race` | `race` (бортов «∞», успей за таймер) — самый сильный одиночный источник сложности. |
| **События** | `events:{}` | Каждый спецборт добавляет вес (`vip .5`, `rush 1`, `medical 1`, …). Максимум одно новое событие на карту и только после L6. |
| **Среда** | погода / де-айсинг | Враждебная среда (снег→де-айсинг, плуг) добавляет **шаг маршрута**, а не таймер ожидания. |
| **Зажим ресурсов** | `startMoney`, меньше открытых `slots` | Затянуть одну карту **без** раздувания числа бортов. Предпочтительная ручка. |

**Правило большого пальца:** чтобы усложнить карту, лучше *зажать ресурсы* или *добавить одно
запланированное событие*, чем раздувать число бортов. Скачки числа ощущаются как гринд (нудная
рутина); давление ресурсами ощущается как головоломка.

## Форма кривой

Макро-наклон **пологий и монотонный** (всё время вверх без провалов), но микро-форма — **пила**:
каждый раз, когда вводится новая механика, окружающая нагрузка *проседает*, чтобы игрок встретил
новую игрушку в спокойных условиях, а потом нагрузка снова подрастает до следующей игрушки.

```
нагрузка
 ▲                                              ╱╲  L10 итог
 │                              ╱╲  час пик    ╱
 │                 ╱╲  VIP    ╱            ╲╱
 │      ╱╲ таймер╱        ╲╱
 │ ╱╲ ╱        ╲╱
 │╱  ╲╱   ← провал на каждом уровне с «новой идеей», затем рост
 └────────────────────────────────────────────► уровень
   L1   L3   L5    L7    L8        L10
```

**Передышки** держат кривую человечной:
- **Бонус-уровни каждый 5-й** (5½, 10½ …) — спокойный «другой мир» на той же механике, без
  событий, с щедрым терпением. Это передышка, а не ступень сложности. Вне кампании.
- **Zen-режим** — routing (проводка бортов) без проигрыша и без давления, всегда доступен.
  Клапан для сброса напряжения.

## Гарантии против фрустрации (пол)

Кривой можно быть вызовом на 3★, но она никогда не должна быть *нечестной* на 1★:

1. **1★ — это проходной порог**, он задаётся достижимым; за него открывается следующий уровень.
   Контент никогда не заперт жёстко за 2★/3★.
2. **Деньги никогда не блокируют звезду.** `levelEconomy()` выводит выплату так, чтобы умелый
   игрок мог позволить себе ожидаемую достройку апрона (`ECON_GEN_BASE ≥ 1` — это гарантия 3★).
   См. [`economy.md`](economy.md).
3. **Хотя бы одна полоса всегда открыта** — помехи/события закрывают максимум одну из ≥3 полос,
   бригады работают параллельно (принцип MVP №1: никогда не убивать ядро игры).
4. **Никаких сюрприз-событий в обучающем блоке** (L1–L6) — сначала игрок учит чистые механики.

## Как продлевать кривую за L10

Новые карты продолжают ту же лесенку; не перезапускай сложность:

1. **Держи «одну новую идею на уровень».** Выбери единственную механику/тип борта, которой
   учит карта (следующая ступень [лесенки меты](meta_progression.md)). Запиши её как вызов
   `level.d.<n>` + подсказку `level.h.<n>`.
2. **Продолжай счёт.** Задай `stars:[s1,s2,s3]` по возрастанию, стартуя около того, где
   закончилась прошлая карта — мягкий рост, скромный спред.
3. **Пиши блоками по ~6** с **capstone** (итоговым уровнем) в конце каждого блока, который
   *совмещает* новые игрушки блока (L10 — первый capstone: VIP + час пик + медицинский разом).
4. **Одно новое событие на карту, никогда не два.** После capstone следующий блок вводит
   очередную семью механик на просевшей нагрузке (пила сбрасывается).
5. **Затягивай ресурсами, а не числом**, когда карте нужно ощущаться труднее без новой механики
   (`startMoney` ↓, открытых `slots` ↓).

Полосы сложности как ориентир: **L1–L6** введение (сложность ~0.06–0.5), **L7–L12** спецборты
(~0.2–0.6), дальше блоки наслаивают среду + стеки событий, но каждую новую идею по-прежнему
открывают на провале нагрузки.

## Зафиксировано тестами

`validateLevels()` (внутри `validateGame()`) следит за инвариантами кривой, чтобы она не «поплыла»:

- `stars` строго по возрастанию; `target = s3` (потолок).
- `race` требует `time`; правила порога `upg`.
- **Нет событий в L1–L6** (граница обучающего блока, `TUTORIAL_LEN = 6`).
- `levelEconomy()` прогоняется для каждой карты и валит сборку, если карта становится
  непроходимой по деньгам (см. [`economy.md`](economy.md#invariants-locked-by-tests)).

После правки любого уровня запусти `npm test` — набор тестов сразу скажет, нарушила ли карта
правило кривой.
