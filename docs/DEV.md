# Документация разработчика (вход для ИИ/нового участника)

Цель файла: чтобы другой ИИ или человек за 5 минут понял устройство проекта и мог
продолжать работу, ничего не сломав.

## Что это
Мобильная игра-диспетчер аэропорта. Полное описание игры, ролей, плана и бэклога —
в [`docs/tower_project_brief.md`](tower_project_brief.md). Правила простым языком —
в [`docs/FAQ.md`](FAQ.md). История изменений — в [`../CHANGELOG.md`](../CHANGELOG.md).
Арт-бэклог (что отрисовать) — в [`assets.md`](assets.md).

## Технологии
- **HTML5 + Canvas 2D + TypeScript** (без фреймворков, без рантайм-зависимостей).
  Исходник **модульный** в `src/`: `src/game/01..13-*.ts` — куски одного IIFE
  (склеиваются по порядку), `src/styles.css`, `src/boot-sw.js`, каркас
  `index.template.html`. `01`/`13` остаются `.js` (открывают/закрывают IIFE).
  **`npm run build`** (`scripts/build.mjs`) снимает типы и склеивает всё в один
  самодостаточный `index.html` — **генерируемый артефакт, в git его нет, руками не
  править**.
- Запуск: `npm run build`, затем открыть `index.html`; либо `npm run serve` (сборка +
  `python3 -m http.server 8123`). Проверка типов — `npm run typecheck`. Прод — GitHub
  Pages, публикует робот (`.github/workflows/deploy.yml`) на каждый push в `main`
  (typecheck + build + тесты).
- Порт на Android/iOS планируется через **Capacitor** (см. бриф).

## Облик: единый скин «неон»
У игры **один облик — `neon`** (тёмный ночной радар со свечением). Выбора скина
больше нет: прежняя мультискиновая система (`cozy`/`neon`/`cartoon`, реестр
`SKIN_DEFS`, правило «логика — родитель над скинами») убрана.

На практике (`src/game/01-bootstrap-theme.js`): неоновая палитра `NEON_TOKENS` накладывается поверх
базовой `PALETTE`; поле рисует `drawNeonField`; спрайты берутся из
`assets/sprites/neon/` (базовые `assets/sprites/planeflow-*.svg` — фолбэк), а
`ATLAS` — просто флаг «спрайт-атлас уже загружен?». Логику, механику и геометрию,
как и раньше, держим независимыми от облика.

## Структура кода (`src/game/01..13-*.ts`)
Игра — один IIFE, **порезанный на модули `src/game/01-…` … `13-…`** (склеиваются
по порядку в один `<script>`; сборка снимает типы). Секции ниже идут в том же
порядке и примерно соответствуют этим модулям:
1. **`PALETTE`, `THEME`, `COL`, `SVC`** — палитра, тема и соответствие услуга→цвет.
   `PALETTE` — базовый источник цветовых токенов. Спрайты в
   `assets/sprites/planeflow-*.svg` хранят цвета как `var(--token, #fallbackhex)`,
   поэтому перекрашиваются на лету: атлас `SPRITES` впрыскивает `<style>` с
   переопределением токенов в каждый спрайт (`SPRITES.setTheme({…})` — глобально,
   либо объект-override в `blitC/blit` — точечно), ключ кэша учитывает подпись
   темы. Активная тема — неоновая (`NEON_TOKENS` поверх `PALETTE`); её несёт
   `THEME.tokens`. `COL` (процедурный фолбэк) читает те же токены через
   `THEME`/`:root`, так что перекраска одинаково ложится и на спрайты, и на фолбэк.
   Подробности — в `assets/sprites/README.md` (раздел «Recolor mechanism»). `SZ`
   (масштаб техники) фиксирован ×1; `ATLAS` — флаг «спрайт-атлас загружен».
2. **i18n: `I18N` / `t()`** — словари строк (en/ru) и весь слой локализации
   (`detectLang/setLang/applyI18n/fmtNum/fmtMoney`). См. раздел «Локализация».
3. **`K`** — все балансные числа (скорости, терпение, деньги, цены, время обслуживания,
   радиусы). **Крутить баланс — здесь.**
4. **`LEVELS` / `LV`** — конфиги уровней (раскладка боксов, число ВПП, `objective`:
   метрика `served`/`upgrades`, градация `stars:[1★,2★,3★]` — `target` выводится
   автоматически = `stars[2]` через `normObjective()`; опц. `time`, `race`, `upg`;
   `events` — какие спецборты/события разрешены: `vip`/`emergency`/`medical`/`rush`/
   `fog`/`wind`; опц. `startMoney`). Схема и кривая сложности — в
   `docs/design/game-design/level-pattern.md`. Спавн и таймеры событий гейтятся через
   `levelEvents()` (в Zen — все спецборты, без динамических). Имён в конфиге нет: имя =
   `levelName()` (`level.t.<n>` с фолбэком на `level.name`); вызов/подсказка —
   `level.d.<n>`/`level.h.<n>`; описание-фолбэк = `objectiveDesc()`. `LV` — текущий
   уровень. `levelIdx`, `levelPassed`, `save`, `SAVE_KEY`, `VERSION`.
5. **`resize()` / `layout()`** — пересчёт геометрии под размер экрана: поле, боксы по
   трём сторонам, ВПП справа, зона зависания. Геометрия в долях экрана.
6. **Состояние** — `planes`, `money`, `lives`, `served`, `gameTime`, `running`,
   `paused`, `inMenu` (в меню на канвасе сцена-радар, не поле), `effects`, `floaters`
   (плавающие «+N ₿»/«−50%»), `lossLog`, `toast`, `debug`.
7. **Звук/вибро: `SND` / `HAP` (v0.12)** — весь звук синтезом Web Audio, без файлов.
   Сейчас активны только три звука: `btn` (тап по меню), `lock` (щелчок фиксации
   маршрута у бокса/ВПП) и `screech` (визг резины на касании ВПП); остальные методы
   `SND` — пустышки (намеренно), включая бип терпения (`alarmAt` всё ещё считается в
   `update`, но звука нет). Вибрация `navigator.vibrate` (Android). Аудио-контекст по
   первому жесту (autoplay-политика). Тумблеры — в настройках паузы, состояние в
   `save.sound`/`save.vibro`. Правило: звук необязателен, всё дублируется визуалом.
   Звук — общий набор: базовые рецепты в `RECIPES`, диспетчер `play(name)`; вызовы
   `SND.*` в `update`/вводе одинаковы для всей игры.
8. **`reset()` / `spawnPlane()`** — старт смены и спавн бортов.
9. **Ввод** (`down/move/up`) — рисование траектории пальцем, тап (стоп/возобновление,
   выделение), тап по боксу (открыть/апгрейд), кнопка паузы.
10. **`update(dt)`** — игровая логика: спавн, движение по зонам, посадка/обслуживание/
   взлёт, терпение (`groundPenalty()` — единая точка наземного штрафа), столкновения,
   уборка, проверка проигрыша.
11. **Рендер (облик «ночной радар»)** — помощники `hexa/starfield/vignette/
   heart/drawIcon/planeShape/drawPlaneBodyAt/bayWalls` + сцены `drawMenuScene` (меню: радар,
   анимация посадки) и `drawField/drawRunways/drawBay/drawPlane/drawEffects/drawFloaters/
   drawHUD/drawToast` (поле). Стиль: тёмный «ink», фосфорные свечения (`shadowBlur`),
   скруглённые панели; кольцо терпения бирюза→янтарь→красный — **только в воздухе**
   (на земле остаток времени показывает карточка борта `drawPlaneCard`). Поле рисует
   `drawNeonField` (через тонкий `drawField`), боксы — `drawNeonBay`. Логика/геометрия/
   расчёты в рендер не уходят.
12. **Экраны и меню** — функции `setPaused/showLevels/startLevel/renderLevels/
    loadGame/saveGame/...` и привязка кнопок DOM. Цикл `frame()` (ветка `inMenu`).
    **Главный экран (`#startScreen`)** собран по хендофу «Main Screen» (облик neon):
    бренд слева по центру, matte-glass панель кнопок справа. Блоки (`.pf-brand`,
    `.pf-cta`, `.pf-chips`, `.pf-corner`, `.pf-scrim`) позиционируются абсолютно.
    Чип звёзд наполняет `updateStartChips()` (заработано/максимум по основным
    уровням), вызывается из `showStart()` на каждом показе экрана. Вертикальное
    центрирование `.pf-cta` — через `margin-block:auto`, **не** `transform`
    (входная анимация `.panel` сбрасывает `transform` в конце и сломала бы `translateY`).
13. **Достижения `ACH`** — самодостаточный модуль медалей: события `ACH.onX(...)`,
    прогресс в `save.ach`/`save.stats`, экран «Медали» (`renderMedals`), тосты.
    Анти-спам (v0.11): ≤2 медалей за раунд (`RUN_CAP`), тосты не чаще раза в минуту
    (`TOAST_GAP_MS`), остаток очереди — сразу на итогах (`flushToasts`); 3-я и далее
    медаль раунда сгорает (одноразовые — до нового совершения действия,
    накопительные придут при следующем срабатывании счётчика), «Легенда» —
    на итогах раунда.

## Ключевые модели (инварианты — не ломать молча)
- **Зоны борта** `pl.zone`: `air` (в воздухе/заходит) → `runway` (на полосе) →
  `field` (рулит) → `bay` (в боксе). Взлёт: снова `runway`.
- **Столкновения только физические:**
  - в воздухе борта **не** сталкиваются (висят/летят, линии можно пересекать);
  - на ВПП и на поле — крушение по близости (`K.CRASH_DIST`) между бортами в той же
    зоне; в боксе — при попытке въезда второго борта.
- **Терпение:** воздушное тикает до посадки; при посадке обнуляется, включается
  наземное (`база + услуги*шаг`). Множитель ожидания: белый ×2, вип ×1.
  Воздушный таймаут → −1 жизнь. Наземный таймаут → штраф −50% (без краша).
- **Очередь услуг:** бокс обслуживает, только если его тип = текущая нужда борта.
  Вылет возможен лишь после всех услуг. Деньги — при вылете, за все услуги (вип ×2),
  с множителем серии `comboMult()` и экспресс-бонусом за быстрый цикл.
- **Звёзды (`computeStars`):** градация по порогам `objective.stars=[1★,2★,3★]` на
  основной метрике (`metricValue()`): звезда за каждый достигнутый порог. Если задан
  `objective.upg`, для 2★/3★ нужны ещё и апгрейды (`upgradesDone`). Уровень завершается
  при достижении потолка (`target`=`stars[2]`), по `objective.time` или по жизням; на
  `race`-уровне — только по времени (потолка по принятым нет).
- **Серия/чистота:** `combo` (растёт за чистый вылет, сбрасывается крушением/штрафом),
  `runCrashes`, `runPenalties` — счётчики за смену (сброс в `reset()`).
- **Спец-борты/события:** `pl.emergency` (малое `airMax`, ×1.5 денег); события по
  таймерам `next*`/`*Until`: «час пик» (ускоряет спавн), «смена ветра»
  (`runway.closed` — посадка/взлёт только на открытые), «туман» (`taxiSpeed` снижен).
  Сейчас «смена ветра» и «туман» **заглушены во всех уровнях кампании** (код механик
  сохранён); активен только «час пик».
- **Карты биомов:** `BIOMES` — отдельная от `LEVELS` ветка карт (каждая — конфиг
  уровня + флаг `LV.biome`); запуск через `buildBiome`/`startBiome`, экран выбора
  `showBiomes`. Свой ключ сохранения `levelKey` (строка `b_<id>`, число у кампании —
  совместимо со старыми сейвами), кампанию биомы не разблокируют. Лесной биом
  (`LV.biome==='forest'`): константы `FOR`, состояние `hazards`/`crews`/`nextHazard`,
  логика `updateForest` (помехи закрывают `runway.closed`, тап → `dispatchCrew` из
  `field.service`), отрисовка `drawForest`/`drawForestDecor`. Свои помехи вместо
  ветра/тумана; в Zen и в туториале не активны.
- **Бонус-уровни** (`BONUS`, флаг `LV.bonus`): шуточный «другой мир» каждые 5 уровней
  («луг бабочек»: борт→гусеница, бокс→цветок, удачный вылет→бабочка 🐛→🌸→🦋). Отдельная
  от `LEVELS` ветка (как биомы): свой строковый ключ сохранения `bonus_<id>` — кампанию
  **не** двигают (`recordResult` двигает прогресс только для числового `levelKey`).
  Запуск `buildBonus`/`startBonus`; в списке уровней карточка «N½» вставляется сразу за
  уровнем N (`bonusCard`), открывается прохождением уровня N на ≥1★ (`bonusUnlocked` —
  читает `save.stars[after-1]` или отладочный `debug.unlockAll`).
  **Своя механика (не как кампания):** гусеница не прилетает, а **вползает** справа по
  земле (`bonusCrawl`, спавн сразу `zone:'field'`, без воздуха/ВПП); несёт **одну** услугу —
  цветок **своего цвета** (3 вида ↔ 3 цвета, `BSP`/`BTYPE`, строгое совпадение через
  существующую проверку `b.type===need`). Стадии в поле `pl.bug`: `cat` → (в цветке) `cocoon`
  (куколка, без маневра — отдельная ветка в зоне `bay`) → `fly` (бабочка упорхивает к правому
  краю и `depart`). Спокойный мир: крушения и near-miss отключены при `LV.bonus`.
  Отрисовка стадий — `drawBug` → `drawCaterpillar`/`drawCocoon`/`drawButterfly` (палитра
  `BSP[species]`); цветок-бокс и пузырёк-цель — `drawFlower` цветом вида. Декор луга —
  `drawBonusDecor` (палитра `BCOL`). Валидатор `validateBonus()` держит инвариант
  «бонус каждые 5 уровней» (`after % 5 === 0`).
- **Zen-режим** (`zen`): отключает потерю жизни, наземные штрафы, крушения, события и
  завершение по цели/времени; `recordResult()` в Zen ничего не пишет.
- **Прогресс:** `save = {unlocked, best, stars, lang, ach, stats, sound, vibro}` в
  `localStorage` (ключ `SAVE_KEY`); ≥1 звезда открывает следующий уровень. «Сбросить
  прогресс» обнуляет только уровни — язык, медали и звук/вибро остаются.

## Тесты
Цель — **страховка под новые механики**: добавляешь уровень / событие / правило —
тесты сразу падают, если сломалось или конфликтует со старым поведением. Не гоняемся
за балансом (числа намеренно крутятся) и за пикселями canvas — тестируем **правила и
контракты**. Три слоя:

1. **Рантайм-валидатор конфига** (`validateGame()` в `src/game/05-validate.ts`: `validateLevels()` +
   `validateI18n()`). Проверяет инварианты: L1 без событий, новое событие только каждые
   3 уровня, валидность боксов/ВПП/целей, паритет ключей i18n между языками. Зовётся
   мягко на старте (пишет в `console.error`, видно в dev), жёстко — из тестов.
2. **Юнит-логика** (`tests/logic.test.mjs`, раннер `node:test`) — гоняется **офлайн без
   браузера** через заглушку DOM (`tests/harness.mjs`): валидатор, гейтинг событий
   (`levelEvents`, Zen), описания целей/имена, звёзды и открытие уровней по реальному
   пути `recordResult()`, загрузка/миграция сейва.
3. **E2E-смоук** (`tests/e2e.spec.mjs`, Playwright + Chromium) — загрузка без ошибок,
   проводка меню/выбора уровня, прогрессия. Нужен браузер (в песочнице на вебе его
   скачать нельзя — гонять на хосте/CI).

Доступ тестов к логике — через **тест-API `window.__GAME`**, который выставляется
**только при `?test=1`** (прод-поведение не меняется). Там же `simulateResult(...)` —
прогон завершения смены без перетаскивания бортов.

```sh
npm test          # слои 1–2 (Node, офлайн) — основная страховка
npm run test:e2e  # слой 3 (нужен: npx playwright install chromium)
```

## Локализация (i18n) — обязательное требование
Игра переводится на другие языки. **Реализовано в 0.9** (долг «строки захардкожены»
закрыт). Правила:

- **Никакого захардкоженного текста в UI и на канвасе.** Любая строка, которую видит
  игрок (кнопки, заголовки, подсказки, toast-уведомления, причины крушений, названия
  уровней/услуг, FAQ-вставки), берётся через слой i18n по **ключу**, а не пишется
  литералом в разметке/коде. **Новые строки — сразу через `t(...)`.**
- **Структура:** таблица строк `I18N[lang][key]` (словари `en`, `ru`) + функция
  `t(key, params)` с подстановкой плейсхолдеров `{n}` (`t('toast.goal', {desc})`).
  Значение-**массив** = формы множественного числа; форма выбирается по `params.n`
  функцией `_plural` словаря (en — 2 формы, ru — 3).
- **Язык:** `en` — основной и фолбэк (нет ключа в текущем языке → берём en → сам ключ).
  Автоопределение `navigator.languages` + ручной переключатель в настройках (пауза →
  Настройки → Язык); выбор хранится в `save.lang` (`localStorage`), сброс прогресса
  его не трогает.
- **Статический HTML** размечен атрибутом `data-i18n="ключ"` — тексты проставляет
  `applyI18n()` (она же — `<title>`, атрибут `lang`, кнопки языков). Динамика (HUD,
  канвас, тосты) зовёт `t()` в момент отрисовки. В `lossLog` хранятся **ключи**
  (перевод при показе — лог переживает смену языка).
- **Числа/деньги** — через `fmtNum()`/`fmtMoney()` (формат по `_locale` словаря);
  время — `fmtTime()` (mm:ss, универсально).
- **Верстка под перевод:** UI должен выдерживать **более длинный текст** (немецкий,
  французский длиннее русского) — не фиксировать ширину под конкретную строку, не
  обрезать. Держать в уме будущий RTL.
- **Добавление языка = добавление словаря** в `I18N` (служебные ключи `_name`,
  `_locale`, `_plural` + все строки) — без правок логики: переключатель и
  автоопределение подхватят его сами.

## Конвенции работы
- **Правим `src/`** (модули `src/game/*.ts`, `src/styles.css`, `index.template.html`),
