# Google Play «Featuring» — квалификация на редакционные подборки

> Разбор гайда https://play.google.com/console/about/guides/featuring/ и статей, на которые
> он ссылается (`developer.android.com/quality`, `growyouraudience`, `releasewithconfidence`,
> `engage-and-retain-your-users`, гайды по стор-листингу и промо-контенту). Это **отдельная**
> от [`play-level-up-plan.md`](play-level-up-plan.md) программа: Level Up — конкретный
> чек-лист UX/PGS-требований для буста в поиске; Featuring — про попадание в **редакционные
> подборки**, оценивается по 4 «столпам качества» + маркетинг-ассетам. Что уже покрыто
> Level Up-чек-листом — здесь только упомянуто, не повторяется.
>
> **Контекст:** как и Level Up, Featuring даёт эффект только **после** публикации, но часть
> пунктов (маркетинг-копия, ASO, ассеты) дешевле закрыть до релиза, пока карточка ещё не
> отправлена на модерацию.

## 1. Core Value

| Требование | Статус в planeflow | Что сделать |
|---|---|---|
| Оригинальность и «полезность/увлекательность» | ✅ есть — механика разбора трафика в потоке, позиционирование через flow-state ([`store-copy.md`](store-copy.md)) | — |
| Соответствие карточки фактическому поведению приложения | 🟡 черновик карточки готов, но реклама/покупки ещё не подключены | Перед публикацией сверить описание и скриншоты — не подразумевать функции, которых пока нет (ads-free формулировки, мультиплеер и т.п.). **Сложность:** низкая. |
| Пиринговый бенчмарк (рейтинг/ретеншен/engagement против похожих игр) | 🚫 недоступно до публикации | После публикации сверяться с Play Console peer benchmarks, не только с абсолютными цифрами. **Сложность:** низкая (после релиза). |

## 3. Technical Quality

| Требование | Статус | Что сделать |
|---|---|---|
| 60 FPS / без крашей на референсных устройствах | 🚧 см. Level Up + [`backlog.md`](backlog.md) («Производительность/рендер») | Доп. аргумент: у Google для игр порог — **минимум 30fps на основной цикл, 60fps+ рекомендован для реакционных игр** — planeflow (драг-реакция) целится в 60, это не «желательно», а профильное требование жанра. |
| ADPF, Game Mode API, frame pacing, Android Performance Tuner, Vulkan/ASTC | 🚫 **неприменимо** | Все — нативные GLES/Vulkan-surface инструменты; planeflow рендерится в Canvas2D внутри WebView (та же логика, что уже применена к Vulkan в Level Up doc). Явно зафиксировано, чтобы не пересматривать. |
| Размер сборки / Play Asset Delivery | ✅ измерено — `www/` (веб-часть AAB) ≈ 17 МБ, с большим запасом от лимита базового модуля **150 МБ**; Play Asset Delivery не требуется | — |
| Время старта (launch-to-interactive) | ⏸️ не измерено | Замерить время от старта Capacitor/WebView до интерактивности (загрузка спрайт-атласа, флаг `ATLAS`); если > ~2 сек — показать индикатор загрузки. **Сложность:** низкая-средняя. |
| «Healthy releases» — фиче-килсвитч без релиза в сторе | 🚫 не реализовано | Firebase Remote Config для рискованных фич (Survival-лидерборд, новые ачивки) — можно выключить без публикации новой сборки. Ложится на уже интегрированный Firebase Analytics. **Сложность:** средняя. |
| Целевой Android API / формат AAB — публикационные требования Play | 🟡 частично — сверено по коду: [`setup-android.mjs`](../scripts/setup-android.mjs) ставит `targetSdkVersion = 36` (Android 16) при `compileSdk 36`, `minSdk 23`; это **выше** текущего минимума Play для новых приложений (API 35 / Android 15 на середину 2026) — с запасом | Значение в коде норм; **минимум Play меняется ежегодно** — пере-сверить `targetSdkVersion` на момент подачи. | Анкета контент-рейтинга (Content rating questionnaire) | 🚫 не заполнена, нигде не отслежена | Добавить как отдельный шаг в чек-лист публикации ([`capacitor-android.md`](capacitor-android.md)) — обязательный пункт Play Console, не связан с кодом. **Сложность:** низкая (ручная). |

## 5. Маркетинговые ассеты и промо (store listing)

> Конкретика ниже уже внесена отдельными правками в [`store-assets-brief.md`](store-assets-brief.md)
> и [`store-copy.md`](store-copy.md) — здесь просто список источника.

- Feature graphic: alt-текст (≤140 симв.), не сплошной белый/чёрный/тёмно-серый фон, цвет
  в тон иконке, без слов «Best/#1/Top/New».
- Скриншоты: alt-текст на каждый, чистый статус-бар (полный заряд/сигнал), подпись ≤20% кадра.
- Скриншоты планшета: **минимум 4** для бейджа «Designed for tablets» (было — «желательно»
  без цифры).
- Промо-видео: отключить рекламу на видео, не возрастное ограничение, встраиваемое (embeddable).
- Локализация текста на скриншотах/feature graphic в RU (сейчас только EN черновик).
- ✅ Короткое описание: `OFF`/`ON`-капс разведён по полям — капс остаётся брендовым хуком
  на feature-graphic, а поле «краткое описание» получило policy-safe вариант без ALL CAPS
  (см. [`store-copy.md`](store-copy.md) → «Краткое описание»).

## 6. Рост и удержание (доп. материал из смежных гайдов)

> Источники: `growyouraudience`, `releasewithconfidence`, `engage-and-retain-your-users`
> (ссылки со страницы Featuring).

- **Пре-регистрация в Play Console** — доступна уже сейчас, до публикации; по кейсам Google
  поднимает day-1 retention/монетизацию. Кандидат в [`backlog.md`](backlog.md) («Потом»).
  **Сложность:** низкая (Play Console, без кода).
- **App Links / deep links** для шеринга результата (счёт/ачивка → ссылка обратно в игру) —
  уточняет пункт бэклога «шеринг результата» из раздела Survival именно как deep link, не
  просто текст. **Сложность:** средняя.
- **Live-ops / сезонный контент** — Google называет это одним из ключевых рычагов удержания;
  напрямую подтверждает уже существующий план
  [`design/game-design/season-leagues.md`](design/game-design/season-leagues.md) — не новая
  идея, но независимое подтверждение с источником.
- **Отвечать на отзывы после публикации** — не код, процессная привычка; Google указывает
  среднюю прибавку **+0.7 звезды** у разработчиков, которые отвечают на отзывы.
- **Push-уведомления для ре-энгейджмента** — с явной оговоркой Google не делать их
  спамными/кросс-промо. Не реализовано и не приоритет сейчас, просто зафиксировано на будущее.
- Store listing experiments (A/B иконки/скриншотов/текста, ≥1 неделя на прогон) и custom
  store listings (до 50 шт., таргетинг по стране/источнику) — обе фичи полезны **после**
  старта трафика. Store Listing Events (промо-анонсы контента) — хорошая точка для будущих
  сезонных дропов, тоже пост-запуск.
- Indie Games Accelerator (менторство от Google) — можно рассмотреть после публикации;
  Indie Games Fund — **неприменим** (гранты только для студий из Латинской Америки).
- Genre insights reports пока не покрывают arcade/puzzle (только MMORPG/Battle Royale/4X) —
  просто иметь в виду, действий не требует.

## Что НЕ применимо (архитектурная причина)

- **ADPF / Game Mode API / frame pacing / Android Performance Tuner / Vulkan / ASTC** — все
  нативные GLES/Vulkan-инструменты; planeflow — Canvas2D в WebView, тот же вывод, что уже
  сделан для Vulkan в [`play-level-up-plan.md`](play-level-up-plan.md).
- **Indie Games Fund** — гранты только для студий, зарегистрированных в Латинской Америке.
- **Engage SDK / подписочные retention-отчёты** — актуально для мульти-приложений/подписок,
  не для одной инди-игры без подписки.
