# Арт-бриф: Store Creative Kit — PlaneFlow

ТЗ на отрисовку **комплекта графики для карточки в сторе** (Google Play): иконка,
скриншоты, feature graphic, промо. Это **не брендбук** — свод правил айдентики живёт в
дизайн-системе PlaneFlow (`assets/sprites/README.md` + [`design/`](design/README.md)).

Смежное: арт-статусы — [`assets.md`](assets.md) (раздел «Для публикации»); упаковка и
чек-лист публикации — [`android-twa.md`](android-twa.md); приватность/Data Safety —
[`play-data-safety.md`](play-data-safety.md).

## Общие правила (для всех ассетов)
- **Облик** — актуальный утверждённый вид игры по **дизайн-системе PlaneFlow**.
  Источники: `assets/sprites/`, спека `assets/sprites/README.md`, дизайн-хаб
  [`design/`](design/README.md). **Конкретные цвета/палитру/тему здесь намеренно не
  фиксируем** — бери актуальные на момент отрисовки (внешний вид может меняться).
- **Единство комплекта** — все ассеты в одном стиле и палитре между собой.
- **Ориентация — горизонтальная** (игра landscape). Скриншоты только горизонтальные.
- **Цветовое пространство** sRGB. Текст крупный, не у самых краёв.
- **Юр. рамка:** только свои ассеты, шрифты и имена; чужое не используем.

## 1. Иконка приложения — полный набор (заменяет устаревшую)
**Зачем:** иконка в карточке Google Play + значок в лаунчере телефона + PWA-установка.
**Что:** один мастер-знак (эмблема PlaneFlow) → экспорт во все варианты ниже. Без текста
и мелких деталей — должна читаться силуэтом на маленьком размере.

| Файл (имена как в репо — чтобы просто заменить) | Размер | Назначение / требования |
|---|---|---|
| `icon-512.png` | 512×512 | **Главная для Google Play** + PWA `any`. 32-bit PNG с альфой, ≤1 МБ, **full-bleed** (без прозрачных углов) |
| `icon-192.png` | 192×192 | PWA `any` |
| `icon-180.png` | 180×180 | apple-touch (iOS/web; не для Play, но в комплект) |
| `icon-adaptive-bg.png` | 512×512 | Android adaptive — **фон во весь кадр** |
| `icon-adaptive-fg.png` | 512×512 | Android adaptive — **эмблема**, прозрачный фон |
| `icon-maskable-512.png` | 512×512 | PWA maskable, full-bleed |
| `icon-maskable-192.png` | 192×192 | PWA maskable, full-bleed |
| `assets/sprites/neon/app-icon.png` | исходник | мастер-исходник иконки |

**Safe-zone (критично — иначе маски обрежут эмблему):**
- **Android adaptive:** холст квадратный — **экспорт 512×512** (спека Android — **108dp**,
  т.е. достаточно **≥432**; 512 — мастер с запасом, тулчейн сам ресайзит под плотности).
  Гарантированно видно только **центральный круг ≈62 %** (≈66dp) — вся смысловая часть
  эмблемы внутри него.
- **Maskable:** safe-zone шире — **центральный круг 80 %**. Чтобы один знак работал и
  там, и там → **держим смысл в более узком круге ≈62 %**, фон заливает весь квадрат.

## 2. Скриншоты телефона — Google Play (обязательны)
**Зачем:** главный продающий элемент карточки. Минимум 2 для публикации, **рекомендуется 4–8**.
**Что:** реальные кадры геймплея/экранов + короткая маркетинг-подпись, **вшитая в
картинку** (у Play нет отдельного поля под подпись).

> ⚙️ База кадров уже генерится автоматически (`npm run screenshots -- --all`, GitHub
> Action «Screenshots»). Задача дизайна — **отобрать/срежиссировать кадры и наложить
> композицию + подписи** (+ опц. рамку-мокап телефона).

**Тех. требования (Google Play, телефон):**
- **JPEG или 24-bit PNG, без прозрачности** (альфа → отклонят).
- Сторона 320–3840 px, соотношение между 16:9 и 9:16, ≤8 МБ каждый.
- Игра горизонтальная → **landscape 16:9, рекомендуется 1920×1080** (можно 2560×1440 для резкости).

**Сюжет 4–8 кадров (предложение):**
1. Суть — ведёшь маршрут пальцем к полосе.
2. Развоз по боксам обслуживания (ремонт / топливо / посадка людей).
3. Аврал / комбо — крючок «ещё один заход».
4. Экран статистики смены.
5. Карта уровней + звёзды.
6. Визуальная кастомизация / темы оформления — если в игре их несколько.
7. События («tiny stories» — VIP / медборт / событие на полосе).

**Подписи:** 1 короткая выгода на кадр, крупный контрастный шрифт системы, отступ от краёв ~10 %.

## 3. Скриншоты планшета — опц., но желательно (бейдж «Designed for tablets»)
**Тех:** категории **7″ и 10″**, landscape, рекомендуется ≥1080p (напр. 2560×1440 или
2048×1536), до 8 шт. на категорию, тот же формат (JPEG / 24-bit PNG, без альфы). Те же
кадры под более широкий холст.

---

## 4. Feature graphic (баннер карточки) — Google Play (обязателен)
**Зачем:** широкий баннер вверху карточки и в промо-местах стора.
**Тех:**
- **Ровно 1024×500 px**, PNG или JPEG, **без прозрачности**.
- Вордмарк PlaneFlow + ключевой визуал игры.
- **Критично по композиции:** ничего важного у краёв (могут обрезаться под разные
  раскладки) и **по центру** — если будет промо-видео, Play накладывает кнопку ▶ по
  центру. Текст крупный, в центрально-безопасной зоне, но не под кнопкой play.
- **Нельзя:** вставлять скриншот целиком или фейковые UI-кнопки (политика Play).

---

## 5. Промо / превью-видео — опц.
- **Промо-видео для Play** — это **ссылка на YouTube** (загрузку файла Play не
  принимает). 15–30 сек, **первые 2 сек — сразу геймплей** (без лого-заставки и чёрных
  полос — политика), горизонтальное.
- **Промо-баннеры вне Play** (соцсети/сайт), если нужны: OG-превью 1200×630, посты под
  Instagram и т.п.

## Смежное — в карточку Play, но это **текст**, не отрисовка
Название (≤30 симв.), краткое описание (≤80), полное (≤4000). Черновик можно взять из
`manifest.json` (`description`). Дизайнеру не рисовать — отмечено для полноты комплекта.
**Слоганы и подписи** (главные + шорт-лист кандидатов) — в [`store-copy.md`](store-copy.md).

## Формат сдачи
Исходники (вектор / Figma / PSD) **+** экспортированные PNG/JPEG в указанных размерах,
**именами как в репо**, разложенные по папкам (`icon/`, `screenshots/phone/`,
`screenshots/tablet/`, `feature/`). sRGB.
