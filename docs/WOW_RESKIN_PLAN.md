- Задача: новый внешний вид (неон, набор WOW) для существующей игры PlaneFlow.
  **Механику и логику НЕ меняем.**
- Статус: **спланировано, код игры ещё не писали.** Это хендофф (план + ассеты в репо).

  **«так же красиво, как рендеры, но не жрать батарею»**; управление — рисование маршрута пальцем.

## 4. Подход (красиво + не жрёт батарею)
Ключ: свечение **уже запечено** в PNG → в кадре НЕ используем `shadowBlur`/живые градиенты (это
главный пожиратель батареи). Тяжёлое рисуем один раз и кэшируем картинкой, дальше — дешёвый блит.
- Растр на основе её PNG, через **промоут per-zone спрайт-пайплайна** в реальный рендер.
- **Фон** — статичный нижний слой, рисуется один раз, блит без перерисовки каждый кадр.
- **ВПП** — спрайт `vpp`, блит по координатам полос (2 шт. справа).
- **Ангары** — рамка как **9-slice** (тянется под размер бэя без искажений) + иконка сервиса по типу.
- **Самолёт** — растеризуем один раз на нужный размер, поворот+блит; типы — **tint** одного спрайта
  (или отдельные ливреи, если оператор даст).
- **HUD** — хром-рамки из WOW-бара (по отсекам), сердца/деньги/таймер рисуем живьём поверх.
- **Батарея:** статический фон отдельным слоем; нет `shadowBlur` в кадре; FPS-троттлинг /
  render-on-demand на статичной сцене; правый размер ассетов под ландшафт-DPI (~2x), оптимизация веса.
- **Ландшафт:** фикс горизонтальной ориентации; согласовать с `ui`-масштабом и вводом
  (рисование маршрута — см. ветку `fix/mobile-taxi-dock`).

- `src/game/02-sprites.ts` — загрузка игровых PNG (промоут per-zone скинов в основной рендер).

- `src/game/01-bootstrap-theme.js`, `src/styles.css` — токены под новый вид; стили HUD-overlay.
- `assets/...` — оптимизированные игровые спрайты из `design/references` (правый размер, 9-slice).
- `index.template.html` / манифест / Capacitor config — лок ориентации landscape.

## 6. Открытые вопросы / допущения (подтвердить перед кодом)
- «батарея/зарядка» среди иконок = арт существующего сервиса (логику не меняем) — **допущение**.
- типы самолётов — `tint` одного спрайта — **допущение** (иначе оператор даёт отдельные ливреи).
- точная горизонтальная раскладка координат (2 ВПП справа, ангары сверху/снизу, очередь в небе справа).

## 7. Проверка
- `npm run typecheck && npm test` — логика/геометрия не сломаны.
- `npm run build:tuning` → `tuning.html` — примерка вида по зонам на живой геометрии.
- `npm run serve` (или Preview) — визуально в ландшафте, скриншоты.
- Батарея: на статичной сцене CPU не крутится (фон не перерисовывается, нет `shadowBlur` в кадре).
- Работа в feature-ветке от `origin/main`; в `main` не коммитить/не мержить без команды.

## 8. Следующие шаги (для продолжения с другой машины)
1. `git fetch && git checkout design/wow-skin` (эта ветка — точка продолжения).
2. Подтвердить допущения (раздел 6).
3. Промоут спрайт-пайплайна (`02-sprites`) + блит-замена в `09`/`09b`.
4. Ландшафт-раскладка координат (`06`/`04`).
5. Нарезать/оптимизировать ассеты `design/references` → `assets/` (размер, 9-slice, tint самолёта).
6. Проверка (раздел 7), скриншоты.
